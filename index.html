How It Works

Two cache levels:

System 1 — Exact match. Fingerprints the execution plan on first run. Identical goal on repeat: served from local SQLite in 2.66ms. Zero tokens. Zero API calls.


System 2 — Semantic match. Similar goal, different inputs. Diffs the stored plan, regenerates only what changed. You pay for the delta, not the full plan.


A background process (Retrospector) quarantines failed segments. A signal bus tracks latency and failure rates per workflow type. The cache gets smarter every run.
